# Code Reading スキル

指定された機能について、フロントエンド・バックエンド横断でコードを読み解き、構造化されたドキュメントを生成する。

---

## インプットの受け取り方

ユーザーから以下のいずれかの形式で対象機能の指定を受ける：

1. **機能名・キーワード**: 「ログイン機能」「決済処理」「ユーザー登録」など
2. **エントリーポイントとなるファイルパス**: 特定のファイルやディレクトリを指定
3. **URL パス・API エンドポイント**: `/api/users` や `/checkout` など

情報が不足している場合は、ユーザーに質問して対象を特定する。

---

## 調査手順

以下の順序でコードベースを調査する：

### 1. エントリーポイントの特定

- ユーザーの指定をもとに、関連するファイルを Grep / Glob で探索する
- フロントエンド（ページ・コンポーネント）とバックエンド（ルート・コントローラー・サービス）の両方を探す
- ルーティング定義、API エンドポイント定義から辿る

### 2. 呼び出しチェーンの追跡

エントリーポイントから、以下のレイヤーを順に追跡する：

- **フロントエンド**: ページ → コンポーネント → hooks → API クライアント
- **バックエンド**: ルーター → コントローラー → サービス → リポジトリ → DB
- **その他**: ミドルウェア、バリデーション、イベント発行、キュー投入など

### 3. 境界の特定

以下の境界を明確にする：

- フロントエンド → バックエンド（API リクエスト）
- バックエンド → データベース（クエリ・ORM）
- バックエンド → 外部サービス（HTTP、SDK、メッセージキューなど）
- バックエンド → ファイルシステム・ストレージ

### 4. 副作用の列挙

コードが外部に対して行う操作をすべて洗い出す：

- DB への書き込み（INSERT / UPDATE / DELETE）
- 外部 API の呼び出し
- メール・通知の送信
- ファイル・オブジェクトストレージへの書き込み
- キュー・イベントの発行
- ログ出力（重要なもの）
- キャッシュの更新・無効化

---

## 出力フォーマット

以下の4セクションで構成する。すべて日本語で出力する。

---

### セクション1: 機能の全体感

機能の目的・ユーザーから見た振る舞い・主要なフローを簡潔にまとめる。

```markdown
## 機能の全体感

**機能名**: <機能名>
**概要**: <1-2文で機能の目的を説明>

### 主要フロー
1. <ステップ1: ユーザーの操作>
2. <ステップ2: システムの処理>
3. <ステップ3: 結果の返却>
```

---

### セクション2: 各コンポーネントの概要

関連する各コンポーネントの役割と、コンポーネント間の関係をアスキーアートで示す。

```markdown
## コンポーネント構成

### コンポーネント一覧

| レイヤー | ファイル | 役割 |
|---------|---------|------|
| Frontend | `src/pages/xxx.tsx` | ページコンポーネント。フォーム表示と送信処理 |
| Frontend | `src/hooks/useXxx.ts` | API 呼び出しと状態管理 |
| Backend | `src/routes/xxx.ts` | エンドポイント定義 |
| Backend | `src/services/xxx.ts` | ビジネスロジック |
| Backend | `src/repositories/xxx.ts` | DB アクセス |

### コンポーネント関連図

+---------------------+     +------------------+     +-------------------+
|  XxxPage            | --> |  useXxx hook     | --> |  apiClient.xxx()  |
|  src/pages/xxx.tsx  |     |  src/hooks/...   |     |  src/api/...      |
+---------------------+     +------------------+     +-------------------+
                                                             |
                                                             | HTTP
                                                             v
                                                     +-------------------+
                                                     |  XxxController    |
                                                     |  src/routes/...   |
                                                     +-------------------+
                                                             |
                                                             v
                                                     +-------------------+
                                                     |  XxxService       |
                                                     |  src/services/... |
                                                     +-------------------+
                                                             |
                                                             v
                                                     +-------------------+
                                                     |  XxxRepository    |
                                                     |  src/repos/...    |
                                                     +-------------------+
```

**注意**: アスキーアートは実際のコードベースの構成に合わせて描くこと。上記は一例であり、テンプレートとしてそのまま使わない。

---

### セクション3: 境界図

フロントエンド・バックエンド・DB・外部サービス間の境界を明示したアスキーアートを描く。各境界で何がやり取りされるかを記載する。

```markdown
## 境界図

┌─────────────────────────────────────────────────────────┐
│  Browser                                                │
│  ┌─────────────┐    ┌──────────────┐                    │
│  │  XxxPage    │───>│  useXxx      │                    │
│  └─────────────┘    └──────┬───────┘                    │
└────────────────────────────┼────────────────────────────┘
                             │ POST /api/xxx
                             │ { field1, field2 }
═════════════════════════════╪══════════════════════════════  ← FE / BE 境界
┌────────────────────────────┼────────────────────────────┐
│  Backend                   │                            │
│                    ┌───────▼────────┐                   │
│                    │  XxxController │                   │
│                    └───────┬────────┘                   │
│                            │                            │
│                    ┌───────▼────────┐                   │
│                    │  XxxService    │──── → 外部API     │
│                    └───────┬────────┘                   │
│                            │                            │
└────────────────────────────┼────────────────────────────┘
                             │ SELECT / INSERT
═════════════════════════════╪══════════════════════════════  ← BE / DB 境界
┌────────────────────────────┼────────────────────────────┐
│  Database                  │                            │
│                    ┌───────▼────────┐                   │
│                    │  xxx_table     │                   │
│                    └────────────────┘                   │
└─────────────────────────────────────────────────────────┘
```

**注意**: 実際のコードベースに存在する境界のみを描くこと。外部サービスがある場合はその境界も追加する。やり取りされるデータ（リクエスト/レスポンスの概要、クエリの種類）を境界線の横に記載する。

---

### セクション4: 副作用の一覧

この機能が外部に対して行う操作を網羅的に列挙する。

```markdown
## 副作用の一覧

| # | 種別 | 内容 | 発生箇所 | 条件 |
|---|------|------|---------|------|
| 1 | DB書き込み | users テーブルに INSERT | `UserRepository.create()` | 新規登録時 |
| 2 | 外部API | Stripe に決済リクエスト | `PaymentService.charge()` | 購入確定時 |
| 3 | メール送信 | 確認メールを送信 | `NotificationService.send()` | 登録完了後 |
| 4 | キャッシュ | ユーザーキャッシュを無効化 | `CacheService.invalidate()` | プロフィール更新時 |
```

---

## 注意事項

- 実際にコードを読んで確認した事実のみを記載する。推測で書かない
- コードベースに存在しないレイヤーやパターンは省略する（例: リポジトリパターンを使っていなければ「リポジトリ」レイヤーは描かない）
- アスキーアートはコードの実態に合わせて柔軟に描く。テンプレートをそのまま流用しない
- ファイルパスは実際のパスを正確に記載する
- 大きな機能の場合は、ユーザーに確認してスコープを絞る
