# コミットメッセージ生成スキル（Conventional Commits）
 
staged changes の内容を分析し、**Conventional Commits v1.0.0** に準拠したコミットメッセージを**日本語**で生成します。
 
## 概要
 
ユーザーは現在 git にステージングされている変更をもとにコミットメッセージを作りたいと考えています。以下の手順で進めてください。
 
1. staged diff を確認する
2. 変更の内容と意図を理解する
3. Conventional Commit 形式のメッセージを日本語で作成する
 
---
 
## ステップ1: staged changes を確認する
 
以下のコマンドを実行してください：
 
```bash
git diff --staged
```
 
出力が空の場合は、ステージングされた変更がない旨をユーザーに伝え、`git add` を先に実行するよう案内してください。
 
あわせてファイル単位のサマリーも確認します：
 
```bash
git diff --staged --stat
```
 
---
 
## ステップ2: 変更内容を分析する
 
メッセージを書く前に、以下を確認してください：
 
- **変更の種類は何か？**（新機能、バグ修正、リファクタ、ドキュメントなど）
- **影響するスコープはどこか？**（モジュール、コンポーネント、設定ファイルなど）
- **後方互換性を壊す変更か？**（APIの変更、機能の削除など）
- **変更の意図は何か？**（なぜこの変更が必要だったのか）
 
---
 
## ステップ3: Conventional Commit メッセージを書く
 
[Conventional Commits v1.0.0](https://www.conventionalcommits.org/) の仕様に厳密に従ってください。
 
### フォーマット
 
```
<type>[任意のscope]: <説明（日本語）>
 
[任意の本文（日本語）]
 
[任意のフッター]
```
 
### ルール
 
- **type**（必須）：下記の一覧から選ぶ
- **scope**（任意）：影響するコードの範囲を括弧内に記述 — 例：`feat(auth):`, `fix(api):`
- **説明**（必須）：**日本語**で簡潔に書く。末尾に句点（。）は付けない
- **本文**（任意）：「何を・なぜ」変えたかを日本語で説明する。1行72文字以内で折り返す
- **フッター**（任意）：`BREAKING CHANGE: <説明>` や `Closes #123` などの参照
 
### コミットタイプ一覧
 
| タイプ | 使う場面 |
|--------|----------|
| `feat` | 新機能の追加 |
| `fix` | バグ修正 |
| `docs` | ドキュメントのみの変更 |
| `style` | フォーマット・空白の調整（ロジック変更なし） |
| `refactor` | バグ修正でも新機能でもないコードの改善 |
| `perf` | パフォーマンス改善 |
| `test` | テストの追加・修正 |
| `build` | ビルドシステムや依存関係の変更 |
| `ci` | CI/CD 設定の変更 |
| `chore` | 雑務・メンテナンス（`.gitignore` の更新など） |
| `revert` | 以前のコミットの取り消し |
 
### 破壊的変更（Breaking Change）
 
後方互換性を壊す変更の場合：
- type/scope の後に `!` を付ける：`feat(api)!: v1エンドポイントを削除`
- フッターに `BREAKING CHANGE:` を追加して詳細を記述する
 
---
 
## ステップ4: 出力形式
 
コピーしてそのまま使えるよう、メッセージをコードブロックで提示してください：
 
```
<生成したコミットメッセージ>
```
 
その後、必要に応じて以下を提案してください：
- `git commit -m "..."` を直接実行するか確認する
- ユーザーのフィードバックをもとにメッセージを調整する
- 無関係な変更が混在している場合は複数コミットへの分割を提案する
 
---
 
## 出力例
 
### シンプルなバグ修正
```
fix(auth): ログアウト時にトークンが更新されるバグを修正
```
 
### 新機能（本文あり）
```
feat(profile): アバター画像のアップロード機能を追加
 
設定画面からプロフィール画像をアップロードできるようにした。
JPEG・PNG 形式、最大 5MB まで対応。
```
 
### 破壊的変更
```
feat(api)!: v1エンドポイントを廃止
 
BREAKING CHANGE: /v1/* のルートをすべて削除した。
MIGRATION.md に記載の /v2/* エンドポイントへ移行すること。
 
Closes #204
```
 
### メンテナンス系
```
chore(deps): eslint を v9 にアップグレード
```
